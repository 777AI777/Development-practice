**画面要件（MVP+WBS）**
**共通方針**
- 用語はBacklog準拠で「課題（チケット）」「子課題（タスク）」「マイルストーン」を使用する。
- 進捗は予定ベースで算出する。実績時間は記録するが進捗には使わない。
- ユーザーは「ユーザー名＋パスワード」のみで個人情報は扱わない。
- スケジュールは「ガントチャート」「一覧ビュー」「未着手一覧（バックログ）」の3画面で管理する。
**画面一覧**
- ログイン/登録
- チーム管理
- プロジェクト管理
- ダッシュボード（全体進捗、今週の実績）
- WBS一覧（課題/子課題）
- ガントチャート
- 一覧ビュー
- 未着手一覧（バックログ）
- 課題/子課題の作成・編集
**ログイン/登録**
目的: ユーザー登録、ログイン、ログアウトを行う。
主要要素: ユーザー名入力、パスワード入力、登録、ログイン、ログアウト、エラーメッセージ。
操作フロー: 登録→ログイン→ダッシュボード。ログアウト→ログイン/登録。
**チーム管理**
目的: チーム作成とメンバー管理、権限管理を行う。
主要要素: チーム一覧、チーム作成/編集、メンバー追加/削除、権限（管理者/一般）、参加日。
操作フロー: チーム選択→メンバー追加/権限変更→保存。
**プロジェクト管理**
目的: チーム内のプロジェクトを作成・編集・削除する。
主要要素: プロジェクト一覧、名称、説明、作成、編集、削除。
操作フロー: プロジェクト作成→WBS一覧へ遷移。編集/削除は一覧から実行。
**ダッシュボード（全体進捗、今週の実績）**
目的: 全体進捗と週次実績を俯瞰し、主要画面へ素早く移動する。
主要要素: 全体進捗（完了した予定時間÷予定時間合計）、今週の実績（完了数/予定消化量）、ガント/一覧切替、未着手一覧への導線、チーム/プロジェクト切替。
操作フロー: チーム/プロジェクト選択→進捗確認→ガントチャート/一覧ビュー/未着手一覧へ遷移。
**WBS一覧（課題/子課題）**
目的: プロジェクト内のWBS階層を把握し、課題/子課題を操作する。
主要要素: マイルストーン、課題（チケット）、子課題（タスク）の階層表示、状態（完了/未完了）、予定開始日/終了日、予定時間、実績時間、優先度、タグ/色、毎日フラグ。
操作フロー: プロジェクト選択→階層展開→完了/未完了切替→作成・編集へ遷移。
**ガントチャート**
目的: 予定開始日/終了日に基づくスケジュールを視覚化する。
主要要素: マイルストーン・課題・子課題のバー表示、期間、一覧ビュー切替、未着手一覧からの割当。
操作フロー: 期間確認→対象選択→作成・編集へ遷移。必要に応じて一覧ビューへ切替。
**一覧ビュー**
目的: 期日順のスケジュールを一覧で管理する。
主要要素: 期日順一覧、状態、予定開始日/終了日、予定時間、優先度、タグ/色、ガントチャート切替。
操作フロー: 期日順確認→対象選択→作成・編集へ遷移。必要に応じてガントチャートへ切替。
**未着手一覧（バックログ）**
目的: 未スケジュールの課題/子課題を優先度順に整理し、スケジュールへ割り当てる。
主要要素: 優先度順一覧、予定開始日/終了日の設定、マイルストーン割当、ガントチャート/一覧ビューへの反映。
操作フロー: 未着手項目選択→日付/マイルストーン設定→ガントチャート/一覧ビューへ反映。
**課題/子課題の作成・編集**
目的: 課題（チケット）と子課題（タスク）の登録・更新・削除を行う。
主要要素: タイトル、状態（完了/未完了）、予定開始日、予定終了日、予定時間（15/30/60分など）、実績時間、優先度、マイルストーン、タグ/色、毎日フラグ、親課題指定、保存、削除。
操作フロー: 各一覧/ガント/未着手から作成・編集→保存→各画面に反映。完了時は簡易アニメーション/称賛メッセージを表示。
**画面遷移の要点**
- ログイン/登録→ダッシュボード
- ダッシュボード→チーム管理/プロジェクト管理/WBS一覧/ガントチャート/一覧ビュー/未着手一覧
- プロジェクト管理→WBS一覧
- WBS一覧→課題/子課題の作成・編集
- ガントチャート↔一覧ビュー
- 未着手一覧→課題/子課題の作成・編集、ガントチャート、一覧ビュー
- すべての画面→ログアウト→ログイン/登録
**要確認事項**
- チームとプロジェクトの関係（1チーム複数プロジェクト前提でよいか）。
- 予定時間の入力単位（15/30/60分固定か、任意入力も必要か）。
- ダッシュボードの集計範囲（チーム単位かプロジェクト単位か）。
- 未着手一覧の「優先度」の定義と編集方法。
- ガントチャート上で日付変更などの直接操作を許可するか。
- デイリータスクの対象と設定場所（子課題のみでよいか）。
- 週次実績の指標の内訳（完了数/予定消化量の両方でよいか）。
