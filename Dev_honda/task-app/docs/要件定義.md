# 要件定義（MVP+WBS）

## 目的
ローカル環境で、複数ユーザーがチーム単位でWBS観点の進捗・予定・実績を一元管理できるタスク管理ツールを作る。

## 用語定義（Backlog準拠）
- マイルストーン: 期間の区切り（例: 企画/実装/テスト）。
- 課題（チケット）: ユーザーに価値が出る機能や作業のまとまり。
- 子課題（タスク）: 課題を実行可能な作業に分割した単位。

## スコープ
- 対象: ローカルで複数ユーザーが利用。
- スコープ内: ユーザー/チーム管理、WBS管理、進捗管理、スケジュール表示、未着手一覧管理、基本タスク操作。
- スコープ外: 外部サービス連携、クラウド同期、通知。

## 機能要件
- ユーザー管理: ユーザー名とパスワードで登録、ログイン、ログアウト。
- チーム管理: 作成、メンバー追加/削除、権限（管理者/一般）。
- プロジェクト管理: 作成、編集、削除。
- WBS階層: プロジェクト → 課題（チケット） → 子課題（タスク）。
- 課題/子課題操作: 作成、一覧、完了/未完了、削除。
- 見積もり: 子課題に予定時間（例: 15/30/60分）を設定。
- タグ/色: 課題・子課題にタグ付けし色で識別。
- デイリータスク: 指定した子課題を毎日未完了に戻す。
- 完了演出: 完了時に簡易アニメーションや称賛メッセージを表示。
- 週次実績: 週単位の完了数や予定消化量の可視化。

## 進捗管理
- 進捗は予定ベースで算出する。
- 指標: 完了した予定時間 ÷ 予定時間合計。
- 実績時間は記録するが、進捗算出には使わない。

## スケジュール表示
- ガントチャート: マイルストーン・課題・子課題を予定開始日/終了日で表示。
- 一覧ビュー: 期日順に並べて表示。
- 未着手一覧（バックログ）: 未スケジュール項目を優先度順に表示。

## 画面要件（概要）
- ダッシュボード: 全体進捗、今週の実績、ガント/一覧切替。
- WBS一覧: 課題/子課題の階層表示。
- 未着手一覧（バックログ）: 優先度順の一覧とスケジュールへの割当。
- チーム管理: メンバー追加、権限変更。

## データ要件（概要）
- ユーザー: ID, ユーザー名, パスワードハッシュ, 作成日。
- チーム: ID, 名称, 作成日。
- チームメンバー: チームID, ユーザーID, 権限, 参加日。
- プロジェクト: ID, 名称, 説明。
- マイルストーン: ID, プロジェクトID, 名称, 予定開始日, 予定終了日, 表示順。
- 課題（チケット）: ID, プロジェクトID, 親課題ID, タイトル, 状態, 予定開始日, 予定終了日, 予定時間, 実績時間, 優先度, マイルストーンID, 毎日フラグ。
- タグ: ID, 名称, 色。
- 課題タグ: 課題ID, タグID。

## 非機能要件
- SQLiteでローカル永続化。
- 起動は `npm install` → `npm run dev` で完結。

## 要確認事項
- WBS階層は「課題（チケット）→ 子課題（タスク）」の2段でよいか。
- 未着手一覧（バックログ）は「優先度順一覧」で十分か、カンバン的な状態列が必要か。
- チーム参加は「管理者が追加」方式でよいか。
