# 要件定義書（Phase分割版 v1.1）

## 1. 背景・目的
- 日常の「好き」がメモやSNSなどに分散し、整理・振り返り・共有がしづらい課題を解消する。
- まずは個人利用で価値を成立させ、その後段階的にSNS化してコミュニティへ発展させる。

## 2. プロダクト方針
- オールジャンル（映画、音楽、旅行先、飲食店、日用品など）でランキングを作成・管理できる。
- SNS投稿を積極的に行わないユーザーでも、個人管理ツールとして価値を感じられる設計にする。
- 将来的には「好き」を軸にしたコミュニティプラットフォームを目指す。

## 3. 対象ユーザー
- 20〜40代のSNSに馴染みがある多趣味層。
- ジャンルを横断して「好き」を整理したいユーザー。
- 発信頻度は高くなくても、自分用に記録・管理したいユーザー。

## 4. 成功条件
### 4.1 ゴール
- 個人のランキングを継続的に作成・管理できること。

### 4.2 KPI/KGI
- KGI: ランキング管理サービスとして継続利用される状態を作る。
- KPI（Phase1）: ユーザー数、DAU。
- 目標値（Phase1リリース3か月後）:
  - ユーザー数: 100人
  - DAU: 20人
- 共有数KPI: 当面設定しない。

### 4.3 判定時期
- Phase1リリース後3か月時点で一次評価を実施する。

## 5. フェーズ計画
### 5.1 Phase1（個人利用フェーズ）
- 目的: 個人で「好き」を整理・保存・更新できる状態を完成させる。
- 主要機能:
  - Webアプリ化
  - ログイン/ログアウト（Google認証）
  - ランキングの作成・編集・削除
  - 最大5位までのランキング
  - 一覧表示
  - 固定タグ（大分類）
  - タグ検索
  - 下書き保存（保持上限: 1ユーザーあたり5件）
  - 削除確認ポップアップ表示
- 公開範囲: 個人利用に限定（他ユーザー公開なし）
- 削除仕様: 物理削除。削除時に「削除すると元に戻せない」旨を表示する。

### 5.2 Phase2（SNS化フェーズ）
- 開始条件: Phase1の実装開発完了後、使用感調査を実施し、結果をもとに開始可否を判断する（数値KPIゲートは設けない）。
- 目的: 共有・閲覧・つながりの機能を追加し、SNSとしての利用価値を拡張する。
- 機能候補:
  - 公開範囲設定（公開/限定公開/非公開）
  - 他ユーザーのランキング閲覧
  - 固定タグの細分類化
  - ログイン方法の追加（Google以外）
  - 通報機能（不適切コンテンツ対策）
  - フォロー、更新通知

### 5.3 Phase3（拡張フェーズ）
- 目的: 利便性向上と事業性検証を進める。
- 機能候補:
  - フリーワード検索
  - 多言語対応
  - リンク/メモ記載
  - 画像登録の課金要素（Phase1対象外、将来検討）

## 6. 優先順位（MoSCoW）
### 6.1 Must（Phase1実装対象）
- 作成
- 編集
- 削除
- 5位までランキング
- Webアプリ化
- ログイン/ログアウト（Google認証）
- タグ検索
- 一覧表示
- 固定タグ（大分類）
- 下書き保存（5件まで）

### 6.2 Should（Phase2以降優先）
- サムネ追加
- 使いやすいUIデザイン
- レスポンシブデザイン
- 100位までランキング
- ランキングのタグ分類（細分類）
- タグの新規追加
- 入力サジェスト
- ログイン方法追加

### 6.3 Could（将来拡張）
- 友人フォロー
- ランキングフォロー/更新通知
- フリーワード
- 多言語対応
- リンクやメモ記載
- 画像登録（課金連携）

## 7. 非機能要件
- セキュリティ:
  - 認証は外部サービス（Google OAuth）を利用。
  - 通信はHTTPSを前提。
- 可用性/運用:
  - 無料枠内でデータ量・コストを監視。
  - リリース後は利用状況を継続モニタリング。
- 拡張性:
  - フェーズ追加を見据え、タグ粒度・公開設定・順位上限を拡張可能な設計にする。

## 8. リスク・依存関係
- リスク:
  - 認証情報およびユーザー情報の取り扱い。
  - SNS化後の不適切コンテンツ対応。
  - 画像対応時のストレージコスト増。
- 依存先:
  - Google認証基盤
  - ホスティング環境
  - データストア
- 回避策:
  - 外部認証の活用。
  - フェーズごとの段階実装。
  - Phase2で通報・モデレーション機能を導入。

## 9. 運用・今後の検討事項
- リリース後戦略（告知・初期流入施策）の具体化。
- 無料枠上限を超えるタイミングの見極め。
- Phase2着手前の使用感調査設計（対象ユーザー、調査項目、評価基準）。
