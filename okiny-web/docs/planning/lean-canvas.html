好きなものを通じて、知らない誰かと静かに繋がれる場所 ｜ 目標：公開12ヶ月で10,000人
今ないもの
  • 映画・音楽・本・カフェなどジャンルをまたいで「好き」をまとめられる場所がない。今はジャンルごとに別々のアプリを使うしかない
  • 「同じものが好きな人が、他に何を好きか」を調べる手段がない。SNSに書いても流れて消える
  • 点数やレビューを書かずに、ただ「好き」を並べるだけで同じ感性の人と出会える場所がない
※2026年4月の調査で、この行動をしている人が大量にいることを確認済み
どう解決するか
  • 「テーマ＋好きなもの3つ」を投稿するだけ
  • 例：「雨の日に聴きたい曲」→ スピッツ / ミスチル / バンプ
  • 投稿が3件たまると好みの傾向分析、5件でポイント消費型の自己分析ツールが解放される（投稿で貯まるポイントを使って詳細分析）
  • おすすめテーマがコミュニティの盛り上がりをさりげなく映す仕掛け
  • スレッド（Q&A）でお題に回答。誰でも自由に作成可能。回答へのAward機能でポイント獲得
他にないもの

「好き」の静かなギャラリー

  • 全ジャンルの好きが1つのプロフィールに蓄積される。これができるアプリは他にない
  • 「〇〇が好きな人は他に何が好きか」をジャンルを超えて検索できる（Phase 2〜）。このデータは世界に存在しない
  • 投稿するたびに自分の好みを再発見できる。3つ選ぶ行為自体が自分との対話になる
真似できない理由
  • ユーザーが増えるほど好みのデータが蓄積され、後から同じものを作っても追いつけない
  • 「好き」しか入力できないので、荒れない空気がルールではなく仕組みで保たれる
誰に届けるか
  • SNSで「おすすめ教えて」「私のトップ3」と日常的に投稿している人
  • 好きなもののコラージュ画像を作ってシェアしている人
  • 映画・音楽・本の記録アプリを複数使い分けている人
  • ラジオやPodcastのリスナー

どうやって届けるか
  • 初期: 投稿カードのSNSシェア＋口コミ＋SNS広告
  • 中期: ラジオ・Podcast番組との連携
  • 長期: Google検索で自然に見つかる状態
かかるお金
  • サーバー代: 初期は無料〜月数千円
  • 広告費: SNS広告でユーザーを集める（1人あたり100〜200円が目標）
  • 運用: お題の企画・通報対応・おすすめ投稿の選定（手動）

初期は赤字前提。本業と並行のため、無理なく続けられる体制。

入ってくるお金（公開6ヶ月目〜）
  • アプリ内広告（Phase 1: AdMobバナー）: 位置制限付きでタイムラインに表示。有料プランで非表示（＝有料化の動機）
  • ポイント課金（Phase 1）: 自己分析ツール利用に必要なポイントを課金で購入可能。投稿でも貯まるが、追加購入で深い分析を即座に利用
  • 有料プラン（Phase 1.5〜）: 好みの詳細分析・振り返り拡張・広告非表示オプション。月額550円
  • スポンサード投稿（Phase 2）: フィードネイティブ広告。体験を損なわない形で企業の投稿を掲出
  • タイアップお題（Phase 2以降）: 企業がお題を出稿。嗜好データ＋プロモーションを兼ねる
  • トレンドレポート（Phase 2以降）: 好みの傾向レポートを企業に販売
  • API＋データ分析（Phase 3〜）: 好みデータの外部提供。成長するほど価値が上がる

ユーザーが楽しむほどデータが溜まり、事業の価値も上がる構造。
