01

コアインサイト

ユーザーリサーチから発見された、OKINYの存在意義の核心。

核心概念

「輪郭欲」

人間は「自分が何者であるか」を確かめるために、好きなものを並べて輪郭を描く欲求を持っている。X（旧Twitter）等の既存SNSではこの欲求が発信欲・承認欲・帰属欲で上書きされており、静かに輪郭を整える場所が存在しない。OKINYはその場所になる。

蓄積の欲
内発的
消えゆく熱量を器に収めたい。起点は「今夜」、ゴールは「半年後の自分」。
制御の欲
内発的
誰に何が届くかを自分で握りたい。日記的な連続性を持つ。
所属の欲
外発的
同じ好きで静かに繋がりたい。他者との遭遇がきっかけで生まれる。

設計哲学

静かな山のみ

OKINYは谷を意図的に設計しない。メトリクスがないSNSにおいて、見返し衝動を人工的に抑制する意味がない。ユーザーは投稿したら、いつでも自分のペースで見返せる。プッシュ通知は対人通知（フォロー・コメント等）に限定し、行動を煽る通知は送らない。ユーザーが自分で戻ってくることに賭ける。

06

デザイン原則 6箇条

全てのUI判断の基準。迷ったらここに立ち返る。

01

騒音つくらない

バッジ禁止。効果音禁止。プッシュ通知は必要最小限（フォロー・コメント等の対人通知のみ）。通知で行動を煽らない。

02

余白は意味持つ

行間1.5倍、画面密度低。空欄3枠そのものが「ここに好きを入れていい」という入口になる。

03

比較の軸持ち込まない

ランク・スコア・いいね数・閲覧数を表示しない。タグの件数も非表示。数字が比較を生む。

04

選んだこと自体が表現

選定理由のコメント欄なし。並べた3つですべてを語らせる。補足説明は蛇足。

05

時間が文脈

半年前の投稿が再発見される導線。更新日による鮮度表現。蓄積されるほど、自分の輪郭が見える。

06

主語をシステムに渡さない

「トレンド」「人気ランキング」でユーザーの選択に介入しない。レコメンドは「事実の提示」として設計し、誘導しない。シェアもレコメンドも、全てユーザーの手元にトリガーがある。

07

タッチポイント全体像

Phase 1（SNSとして成立する最小セット）〜Phase 3で段階的にタッチポイントが増えていく。各機能の意味と、なぜそこにあるかを解説する。

Phase 1（SNSとして成立する最小セット）
1

ホーム・おすすめテーマ

Phase 1

アプリを開いたときの最初の画面。運営発テーマ2-3本とユーザー発テーマ3-4本を合わせた計5-7枚のカードが表示される。たとえば「最近心に残った映画3本」「雨の日に聴きたい曲3つ」など。テーマがあることで「何を投稿しよう」と迷わずに済み、投稿のハードルを下げる。ユーザー発テーマの条件は直近30日で5件以上の投稿があるテーマ。週次でランダムシャッフルされ、ラベル（運営/ユーザー）やカウントは非表示。固定スケジュール配信ではなく常時表示。

蒼井にとっては着火のきっかけ。大澤にとっては「何を書けばいいかわからない」を解消する入口。
蒼井：着火の入口 大澤：投稿ハードルの解消 仮説3-5 テーマが習慣をつくるか 差戻し：固定スケジュール配信→混合推奨モデル
2

投稿フォーム3枠

Phase 1

OKINYの核心。テーマを決め、好きなものを3つだけ入力する画面。空欄が3つ並んでいるだけのシンプルなフォーム。順位なし、スコアなし、レビュー欄なし。「空欄そのものが入口」というデザイン原則②の体現。1投稿 = テーマ + 好きなもの3つ。

3つという制約が「何でも書ける自由」より「厳選する楽しさ」を生む。これがOKINYの自己対話体験の出発点。
蒼井：蓄積の欲＝熱量を器に収める 仮説3-1 3つ固定が最適か 仮説3-2 順位なしでハードル下がるか
3

投稿プレビュー

Phase 1

投稿を確定する前に、完成した投稿がどう見えるかを確認する画面。「この3つでいいか」を自分に問い直す最後の瞬間。プレビューがあることで投稿の丁寧さが担保され、「間違って投稿してしまった」不安を取り除く。

堀川：粒度へのこだわりを担保 離脱リスク：投稿ミスへの不安
4

投稿完成画面

Phase 1

投稿が完了した瞬間に表示される画面。派手なアニメーションや「おめでとう！」は一切なし。完成した投稿が静かに表示されるだけ。これが「投稿できた」という安堵になる。熱量を器に収め終えた直後の、静かな達成感。

蒼井：投稿完了の安堵 原則① 騒音つくらない
5

マイページ

Phase 1

自分が作った投稿の一覧。時系列で並び、過去の投稿をいつでも見返せる。これがOKINYの「蓄積」の実体。投稿が増えるほど自分の「好きの輪郭」が見えてくる。蒼井のゴール「半年後の自分に届く手紙」は、この画面で実現される。

デザイン原則⑤「時間が文脈」の中核画面。半年前の投稿を見返すと「自分はこういう人だったんだ」と気づける。
蒼井：半年後の自分に届く手紙 堀川：自分の変化を追う日記 仮説4-2 選ぶ行為が自己対話になるか
6

投稿詳細

Phase 1

1つの投稿を開いたときの画面。テーマと3つの好きが静かに並ぶ。いいね数も閲覧数も表示しない。他人の投稿を開いたときも同じ見え方。「この人はこの3つを選んだんだ」という事実だけが伝わる。シェア機能はこの画面のオーバーフローメニュー（三点リーダー）内に格納。

蒼井：見返し＝自分の投稿を見返す 大澤：他者の投稿を静かに覗く 原則③ 比較の軸持ち込まない
7

タグ検索結果

Phase 1

タグで投稿を検索した結果画面。たとえば「スピッツ」で検索すると、スピッツを3つの中に入れた人の投稿が一覧で出る。並び順は最終更新日降順で、毎週ランダムにシャッフルされる。人気順やおすすめ順はない。トレンドタグの表示もない。件数も非表示。「静かに探して、見つける」体験。

大澤にとっては「同じものが好きな人がいる」と知る瞬間。OKINYの「所属の欲」はここから始まる。
大澤：似た投稿発見＝静かな所属感 原則⑥ 主語をシステムに渡さない（非誘導検索）
8

テーマ下部・他の投稿リンク

Phase 1

おすすめテーマの下部にランダムで5つの投稿へのリンクが静かに並ぶ。全体統計は表示しない。数字ではなく「他にも投稿がある」という気配だけを伝える。週次で入れ替わる。コミュニティの存在感を、数値ではなく存在そのもので示す設計。

大澤：孤独じゃないと知る気配 仮説4-3 少人数が密度になるか ユーザー発テーマがコミュニティの気配を担う
9

自己分析ツール（ポイント消費型）

Phase 1

投稿やスレッド回答で貯まるポイントを使い、ユーザーが自分で選んで分析を実行するモデル。閾値型（投稿N件で自動解放）ではなく、ポイントを消費して能動的にアンロックする設計。バッジ・トロフィー・進捗バー・達成言語・他ユーザー比較は一切禁止。全段階（3/5/10/20/50件）の分析メニューをPhase 1から提供。分析メニュー案: タグTOP3（5pt）、カテゴリ分布（10pt）、時間帯ヒートマップ（15pt）、タグ変遷（25pt）、好みDNA（40pt）、年間振り返り（50pt）。※分析メニューの詳細・ポイントコストは今後詰める。

デザイン原則①「騒音つくらない」を守りつつ、蓄積に対する静かな報酬を返す。ドーパミン型の達成感ではなく、セロトニン型の「ああ、溜まってたんだ」という気づき。ポイントを「何に使うか選ぶ」行為そのものが自己対話になる。
蒼井：蓄積への静かな報酬 仮説4-1 自己分析ツールが嬉しいか 差戻し：閾値型→ポイント消費型に変更
9b

ポイントシステム

Phase 1

自己分析ツールの消費通貨となるポイントの獲得・消費・課金の仕組み。獲得方法: 投稿1pt、スレッド回答1pt、ピン止めされると+3pt。消費先は自己分析ツールの各メニュー。ポイントの課金購入あり（レート未定）。プレミアム会員はポイント獲得速度2倍。ポイント残高はマイページに静かに表示し、ランキングや他者比較には使わない。

ポイントの存在自体が「投稿する理由」になる。ただし、ポイントを前面に押し出さない。あくまで自己分析を使いたくなったときに「ああ、投稿してたから使えるんだ」と気づく設計。
蒼井：投稿の蓄積がポイントとして可視化される 収益：ポイント課金（Phase 1から） 原則① 騒音つくらない（ポイント獲得通知なし）

外部シェア導線

Phase 1

投稿詳細画面の三点リーダーメニュー内に「シェア」ボタンを配置（副動線）。タップすると、モノクロームカードデザインの投稿カード画像が生成され、各種SNSにシェアできる。デフォルトは限定URL（リンクを持っている人だけ閲覧可能）で、SNSへの公開シェアは追加タップが必要。文言制限あり、誘導文言は使わない。

事業計画書の「投稿カードのシェアが集客の核」を、デザイン原則⑥と両立させた形。ワンタップシェアは廃止し三点リーダー内の副動線に変更。シェアのトリガーがユーザーの手元にある限り、原則違反にならない。
堀川：公開範囲を自分が握るシェア 仮説3-4 投稿カードをシェアしたいか 成長KPI：MAU2000達成のオーガニック経路

AdMobバナー広告

Phase 1

Phase 1の収益源はAdMobバナー + ポイント課金 + プレミアムプラン（月額550円）の三本柱。表示位置に制限あり（マイページ・投稿詳細には表示しない）。プレミアムプランの特典に「広告非表示」を含む。Phase 2でスポンサード投稿（フィードネイティブ広告）を追加。

ネイティブ広告（スポンサード投稿）はPhase 2。Phase 1ではAdMobバナー + ポイント課金 + プレミアムプランに留める。
収益：Phase 1 AdMob+ポイント課金+プレミアム550円 / Phase 2 +スポンサード投稿

投稿時レコメンド

Phase 1

投稿完了時に「○○を挙げた人はこんなものも好きです」を表示。投稿→発見→次の投稿、という「好きの連鎖ループ」を回すエンジン。データ不足時は表示しない（投稿数が少ない初期は非表示）。SNSとして成立するために、初期から好きの連鎖体験を提供する。

デザイン原則⑥「主語をシステムに渡さない」との整合：「おすすめ」ではなく「事実の提示」（○人が挙げている）として設計する。
蒼井：投稿→発見→次の投稿の連鎖 仮説4-6 レコメンドが継続の核になるか
13

スレッド機能（知恵袋型Q&A）

Phase 1

QA型ギャラリー。投稿主がお題を立て、他のユーザーが回答（3つの好き + ひとこと任意140字）をぶら下げる形式。スレッドは誰でも自由に作成可能（ポイント消費なし）。荒らし・ボット対策は別途設計する。対話は不可——投稿主はリアクション（3種: 参考 / 新発見 / 共感）のみ返せる。ピン止め機能あり（投稿主のみ、複数可）。回答はマイページに投稿として蓄積される。運営スレッドも用意し、公式お題で参加ハードルを下げる。

回答の時間差表示
意図: 回答者の独立思考を促進する。先行回答へのアンカリング（係留効果）を防ぎ、回答の多様性を確保する。

仕組み: 投稿後一定時間、他の回答が見えない。1〜3番目の回答は5分間、4番目以降の回答は10分間、他者の回答が非表示になる。スレッド主（お題を立てた人）は常時すべての回答を閲覧可能。

心理学的根拠: 認知的反射の研究において、回答までの遅延時間が増えるほど熟慮的な回答が増えることが確認されている。先行回答が見える状態では模倣や同調が起きやすく、回答の多様性が損なわれる。特に「好き」という個人の感性に関わる領域では、他者の回答を先に見ることで「自分もそれにしよう」「自分の答えは変かも」という心理が働きやすい。時間差表示はこの影響を構造的に排除する。

効果: 「それぞれが自分の中から出す」構造になり、スレッド主が多様な視点のアイデアを受け取れる。同じお題でも十人十色の回答が集まることで、スレッドが「静かなギャラリー」として機能する。これはOKINYの「静かなギャラリー」思想——システムが誘導せず、ユーザーが自分で選ぶ——とも整合する。

UX配慮: 回答投稿後に「皆さんが独立して考えるため、最初は他の回答が見えません」等の説明を表示する。カウントダウン表示で待ち時間を可視化し、「いつ見えるのか」の不安を取り除く。待ち時間終了後に「他の回答が見えるようになりました」と静かに通知する。
「教えたい欲」と「知りたい欲」を静かに噛み合わせる。対話不可・リアクション3種のみという制約が荒れない場を作る。ピン止めされると+3ptで、回答者のモチベーションにもなる。SNSとして成立する最小セットの一部として、Phase 1から導入する。
大澤：「教えてもらう」から参加が始まる 仮説3-6 対話不可+リアクション限定で荒れない場を作れるか
13b

Award機能

Phase 1

スレッドの回答に対してAwardを贈れる機能。ピン止め（投稿主が「参考になった」と示す行為）やリアクション（参考 / 新発見 / 共感）とは別の仕組みとして、回答の価値をより明確に評価・報酬する手段。Award受賞者はポイントを獲得できる。

ピン止めは投稿主の「目に留まった」印、リアクションは軽い感情表現。Awardはそれらを超えた「この回答に価値がある」という意思表示。回答者にとっては投稿のモチベーションとなり、ポイント獲得を通じて自己分析ツール等の利用にも繋がる。※Award の種類・ポイント額・贈呈条件等の詳細は今後詰める。
大澤：回答が評価される静かな報酬 蒼井：良い回答を見つけたときの表現手段 仮説：Awardがスレッド回答の質と量を向上させるか
14

今日のお題

Phase 1

毎日1つ、運営が出すお題。「今日食べたい朝ごはん3つ」「元気が出る曲3つ」のような軽いテーマを日替わりで配信する。おすすめテーマが「常時表示の選択肢」なのに対し、今日のお題は「今日だけの一期一会」。日替わりの制約が「今日書かなきゃ」という軽い動機を生み、投稿の習慣化を促す。プッシュ通知（オプトイン）で配信。

おすすめテーマとの住み分け: おすすめテーマは選んで書く、今日のお題は出されたものに応える。蒼井は「着火のきっかけ」、大澤は「何を書けばいいか迷わない」安心感。
蒼井：日替わりの着火きっかけ 大澤：迷わず書ける安心感 仮説：日替わりお題が投稿頻度を上げるか
15

プッシュ通知

Phase 1

必要最小限のプッシュ通知。対象: フォローされた、コメントがついた、スレッドに回答があった、今日のお題配信。全てオプトイン（初期OFF）で、ユーザーが自分で選んでONにする設計。通知で行動を煽らない。「気づかせる」のみ。デザイン原則①「騒音つくらない」を守りつつ、SNSとして必要な対人通知を提供する。

大澤：フォローやコメントに気づける 原則① 騒音つくらない（全通知オプトイン、初期OFF）
16

フォロー / フォロワー

Phase 1

気になるユーザーをフォローし、その人の新しい投稿をフィードで受け取る機能。フォロワー数は非公開（自分のみ確認可能）。他者のフォロワー数は表示しない。「この人の好きをもっと知りたい」という静かな関心を、数字の比較なしに実現する。

大澤：似た好きの人を静かに追える 堀川：見てくれる存在の可視化 原則③ 比較の軸持ち込まない（フォロワー数非公開）

コメント

Phase 1

投稿に対してコメントを残せる機能。投稿主がコメントの表示/非表示を制御できる。コメント数は非表示。「いいね」の代わりに、言葉で静かに反応を返す仕組み。荒らし対策としてブロック・ミュートと連動。

大澤：言葉で静かに繋がる 堀川：コメントの表示/非表示を自分が握る

ブロック / ミュート

Phase 1

不快なユーザーをブロック（相互非表示）またはミュート（自分に非表示）する機能。SNSの安全性を担保する基本機能。静かな場所を守るために、Phase 1から必須。

堀川：制御の欲＝不快な関係を遮断できる 安全性：SNSの最小要件

ブックマーク

Phase 1

気になった他者の投稿を保存しておく機能。ブックマーク数は投稿者にも他のユーザーにも表示しない。「いいね」ではなく「自分のために取っておく」行為。ブックマークは完全にプライベート。

大澤：気になる投稿を静かに保存 原則③ 比較の軸持ち込まない（ブックマーク数非表示）

OAuth認証

プレミアムプラン

Phase 1

月額550円の有料プラン。特典: 広告非表示、全分析メニュー解放（ポイント不要）、ポイント獲得2倍速、年間振り返りレポート無料、投稿カードデザイン5種追加、下書き上限15件（通常5件）。OKINYの世界観を気に入ったユーザーが「この静けさにお金を払う」という構造。広告密度を上げて無理に誘導するのではなく、価値を理解してくれる人からだけ受け取る形。

「〜が好きな人が好きなもの」検索

Phase 2

OKINYの最大の差別化機能。「スピッツが好きな人は、他にどんな映画を観て、どんなカフェに行くのか？」に答えられる横断検索。蓄積されたデータを使って、あるアイテムを好きな人が他に何を挙げているかをジャンルを超えて表示する。この検索結果は、他のどのサービスにも存在しないデータ。

大澤のジャーニー「重ねる」フェーズの核心。タグ検索から一歩進んで、「好きの繋がり」を可視化する。十分なデータ蓄積が前提のため、Phase 2で有効化。
大澤：好きの繋がりで居場所を深める 事業価値：世界に類のないデータ基盤の実体化

外部DB連携サジェスト

Phase 2

蒼井：入力の手間を減らし、着火の速度を上げる 仮説棚卸し8-3 フリーテキスト表記揺れリスクの解消 横断検索・レコメンドの精度基盤
