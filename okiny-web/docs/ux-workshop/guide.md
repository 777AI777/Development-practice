1人・1週間でStep 1〜7を完走する。
作業は全てAIが実行し、人間は判断・意思決定・インタビューのみ行う。

前提：Step 0（仮説棚卸し）は完了済み。成果物「okiny_hypothesis_inventory.md」を随時参照する。

## Day 1：Step 1 ユーザーリサーチ

### 目的
プロトペルソナの素材を集める。完璧なデータではなく「判断できる程度の手がかり」を得る。

### 手順

1. 自分インタビュー
   以下の質問に自分で書いて答える。正直に、建前なしで。
   - 最後に「好きなもの」をSNSに投稿した（またはしたいと思った）のはいつ、何がきっかけだったか
   - そのとき何を使ったか、なぜそれを使ったか
   - そのとき「こうだったらよかったのに」と思ったことはあるか
   - 「好きな映画3つ」を今選んでみて。選ぶ過程で何を感じたか
   - その3つに順位をつけろと言われたらどう感じるか
   - 5つにしていいと言われたら楽になるか、それとも3つのほうがいいか
   - 200人しかいないSNSに自分なら参加するか？何があれば参加するか

   → 書いたものはそのまま保存。Day 2のペルソナ素材になる。

2. インタビュースクリプト確認
   AIが生成した叩き台を確認。違和感がある箇所だけ修正。

   スクリプト構成（1人20〜30分）：

   【導入 2分】
   「趣味や好きなものをどうやって楽しんでるか聞きたい。正解はないので思ったまま話して」

   【行動を聞く 10分】
   - 普段、好きなもの（映画・音楽・本・カフェ等）の情報をどこで見てる？
   - それについてSNSに投稿したことある？あるなら最近いつ、なぜ？
   - 好きなものの記録ってしてる？してるならどうやって？してないならなぜ？
   - 「○○好きな人と繋がりたい」と思ったことは？そのとき何をした？

   【体験してもらう 5分】
   - 「好きな映画（または音楽・本）を3つ、順位なしで選んで」と頼む
   - 選ぶ過程を観察。迷い方、時間のかかり方を見る
   - 選んだ後に聞く：「選んでみてどうだった？」「3つで足りた？」「順位つけてと言われたら？」

   【コンセプト反応 10分】
   以下を口頭で説明する（1分以内）：
   「好きなものを、テーマと一緒に3つ並べるだけのアプリがある。順位なし、レビューなし。蓄積されてプロフィールになる。同じものが好きな人が他に何を好きかが見える」
   - 第一印象はどう？
   - 今のやり方（X、Filmarks等）をやめてこっちに移ると思う？なぜ・なぜそうしない？
   - 何があったら毎週使いたいと思う？
   - 200人しかいなかったら参加する？

   【締め 2分】
   「他に何か気になったこと、言い残したことがあれば」

3. 知人への連絡
   1〜2人にメッセージを送り、今日か明日中に20〜30分話せるか聞く。

4. SNS観察の検索ワードリスト確認
   以下をX・Threads・Instagramで検索し、投稿者を観察する準備。
   - `#私を構成する42枚`
   - `#今年のベスト映画` `#今年のベスト音楽` `#今年のベスト`
   - `#映画好きと繋がりたい` `#音楽好きと繋がりたい`
   - `#Topsters` `#TierMaker`
   - 「好きなもの まとめ」「おすすめ 記録」

5. SNS観察
   上記の検索ワードで投稿を見て回る。以下をメモ：
   - 投稿者の推定属性（年齢層・性別・ジャンル傾向）※プロフから推定
   - ジャンルの幅（1ジャンル特化か、横断的か）
   - 投稿頻度（1回きりか、定期的か）
   - 不満・要望の発言があるか
   - 「この人に話を聞きたい」と思ったらリスト化

   目標：15〜20人分のメモ。深掘りは不要、パターンが見える程度。

6. 知人インタビュー実施（1〜2人）
   スクリプトに沿って実施。録音許可を取る。
   終わった後すぐにメモを書く（記憶が新鮮なうちに）：
   - 一番印象に残った発言
   - 想定と違った反応
   - コンセプト説明後の第一声

### AIがやること
- インタビュースクリプト生成（完成版を出す。人間は確認して微修正のみ）
- SNS観察記録テンプレート生成
- 検索ワードリストの生成
- インタビューメモを渡せば構造化・整理

### 人間がやること
- 自分インタビューへの回答（自分の内面は自分しか書けない）
- SNSの実際の閲覧と観察（リアルタイムSNSはAI不可）
- 知人への連絡とインタビュー実施（対人は人間のみ）
- インタビュー中の「ニュアンス」の記録（言葉にならない反応の観察）

### Day 1の成果物
- [ ] 自分インタビュー回答（書き出し）
- [ ] SNS観察記録（15〜20人分メモ）
- [ ] 知人インタビューメモ（1〜2人分）
- [ ] SNS経由で「話を聞きたい人」リスト（あれば）

## Day 2：Step 1完了 → Step 2 ペルソナ＋共感マップ

### 目的
Day 1の素材をペルソナに昇華する。

### 手順

1. 全データを並べる
   Day 1の成果物を全部広げる：
   - 自分インタビュー回答
   - 知人インタビューメモ
   - SNS観察記録

2. パターン分け
   全データを見て「OKINYを使う動機の違い」でグループ分けする。
   属性（年齢・性別）ではなく**動機**で分ける。

   分け方の視点：
   - 「自分の好きを整理して振り返りたい」（内向き・記録/自己理解）
   - 「好きなものを誰かに見せたい・反応が欲しい」（外向き・共有/表現）
   - 「同じ趣味の人を見つけたい・新しい好きに出会いたい」（発見/繋がり）

   これはあくまで仮の軸。データから別の分かれ方が見えたらそちらに従う。
   2〜3グループに収める。4つ以上には分けない。

3. 仮説棚卸しとの突合
   Step 0で立てた仮説と、実際に聞こえた声を比較する。
   特に以下の最重要仮説について、確認できたか・覆ったか・不明のままかを記録：
   - 3つ固定が心地よいか
   - 順位なしでハードルが下がるか
   - 本質欲求は「好きへの再接続」か
   - Xで十分ではないか

4. プロトペルソナ記述（2〜3人分）

   各ペルソナに書く項目：

   【基本情報】
   - 名前（架空）・年齢・職業・居住地
   - 趣味とジャンルの幅（例：音楽メイン、映画もたまに）
   - SNS利用状況（何をどのくらい使っているか）

   【行動パターン】
   - 好きなものの記録・共有をどうやっているか
   - 今使っているツール（X、Filmarks、Topsters等）と使い方

   【動機と不満】
   - OKINYに期待すること（何があったら使いたいか）
   - 今のやり方の一番の不満
   - OKINYを使わない理由になりそうなこと ← 必ず書く

   【一言で表すキーワード】
   - このペルソナを象徴するフレーズ（例：「好きなものが散らばってて、全体像が見えない」）

5. 共感マップ作成（各ペルソナ分）

   4象限で整理：
   - **言っていること** — インタビューで実際に出た発言を引用
   - **考えていること** — 発言の裏にありそうな本音
   - **やっていること** — 実際の行動（SNS投稿、アプリ利用等）
   - **感じていること** — 感情（満足、不満、諦め、期待等）

   データが薄い象限は「※仮説」と明記して埋める。空欄で残さない。

### AIがやること
- Day 1のデータを渡せばパターン分けの案を提示
- 仮説棚卸しとの突合チェック（仮説一覧と照合して差分を出す）
- ペルソナの文章化（箇条書きメモ→整った記述に変換）
- 共感マップの4象限テンプレート生成＋データからの記入案
- 「使わない理由」の候補リスト提示（人間が取捨選択）

### 人間がやること
- パターン分けの最終判断（「この人とこの人は同じグループか」の解釈）
- 共感マップの「考えていること」欄の最終判断（行間を読む作業）
- AIが出したペルソナ案の「ここは違う」の修正指示

### Day 2の成果物
- [ ] パターン分け結果（2〜3グループの定義と各グループに該当する人）
- [ ] 仮説検証メモ（最重要仮説4つの現状判定）
- [ ] プロトペルソナ（2〜3人分の記述）
- [ ] 共感マップ（各ペルソナ分）

## Day 3：Step 3 ゴール設定 → Step 4 シーン理解

### 目的
ペルソナごとに「OKINYを使った結果どうなっていてほしいか」を描き、その人がOKINYに触れる場面を立体的に理解する。

### 手順

1. ペルソナ別ゴール記述

   各ペルソナについて以下を書く：

   【ゴール状態】
   OKINYを3ヶ月使った後、この人はどうなっているか？
   - 行動の変化（例：毎週1回はお題に答えている）
   - 感情の変化（例：「自分ってこういう趣味の人だったんだ」と感じている）
   - 他サービスとの関係（例：Xでのマイベスト投稿をやめてOKINYに一本化）

   【提供価値の優先順位】
   このペルソナにとって、以下のどれが一番刺さるか。1位だけ決める：
   a. 好きの整理・蓄積（記録）
   b. 自分の好みの再発見（自己理解）
   c. 同じ好きの人との繋がり（発見）
   d. 好きを外部に表現する（共有）

   ペルソナごとに1位が違うのが正常。全員同じならペルソナの分け方が甘い。

2. 提供価値の矛盾チェック
   ゴールがOKINYの仮説棚卸しの思想と矛盾していないか確認。
   例：ペルソナAのゴールが「DM等で人と深く繋がる」だとしたら、OKINYのDM排除方針と衝突する。これは見て見ぬふりをせず記録する。

3. 利用場面の洗い出し

   OKINYとの接点を時系列で列挙：
   - 認知する場面（どこでOKINYを知るか）
   - 登録を決める場面
   - 初回投稿する場面
   - 2回目以降に戻ってくる場面
   - シェアする場面
   - 離脱しそうになる場面

4. 各場面を6観点で分析（主要4〜5場面）

   書籍Step 3の6観点を使う：

   **① 出来事・タイミング**
   その場面はどんな状況で起きるか。時間帯、曜日、何かのイベントの後か。
   例：金曜の夜、映画を観終わった直後

   **② 役割・影響の意識**
   どんな立場で（趣味人として、友人として等）、それをすることで誰にどんな影響があると思っているか。
   例：友人への「これ良かったよ」の共有、自分の記録

   **③ 人の目**
   その行動をするとき、他人の目をどの程度意識するか。
   例：Xなら数百人に見られる意識。OKINYなら200人の静かな場。

   **④ 物理的環境**
   どこで、どんなデバイスで、通信環境は。
   例：自宅のソファ、スマホ、Wi-Fi

   **⑤ 制約条件**
   時間、予算、ルールに縛られるか。
   例：寝る前の10分。課金はしたくない。

   **⑥ 期待値・先入観**
   「こうなるだろう」という事前の想像。
   例：「SNSだからフォロワー数が大事なんでしょ」「またレビュー書かされるやつ？」

5. 場面ごとのシナリオ文章化
   6観点で分析した結果を、短い物語として書く。
   例：「金曜の夜22時、映画を観終わったAさんは、余韻に浸りながらスマホを開く。Xに書こうかと思うが、前に書いた感想が流れて誰にも見られなかったことを思い出す。そこでOKINYの通知に気づく。今日のお題は『最近心に残ったもの3つ』——」

   この文章がStep 5のジャーニーの素材になる。

### AIがやること
- ペルソナ別ゴールの文章化（ペルソナデータを渡せばゴール案を生成）
- 矛盾チェック（仮説棚卸し×ゴールの自動照合）
- 利用場面リストの叩き台生成（OKINYの機能一覧から場面を導出）
- 6観点分析シートのテンプレート＋各観点の記入案
- シナリオ文章の生成（メモを渡せば物語に変換）

### 人間がやること
- 提供価値の優先順位の最終判断
- 矛盾を「許容するか・修正するか」の意思決定
- 6観点の「期待値・先入観」の最終判断（ユーザーの頭の中の想像）
- AIが生成したシナリオの「リアリティがあるか」の判定

- [ ] ペルソナ別ゴール（2〜3人分）
- [ ] 提供価値の優先順位（ペルソナ別の1位）
- [ ] 矛盾チェックメモ
- [ ] 主要場面のシーン分析シート（4〜5場面×6観点）
- [ ] 各場面のシナリオ文章

## Day 4：Step 5 カスタマージャーニー

### 目的
ペルソナごとに体験の「あらすじ」を時系列で描き、感情の山と谷を特定する。書籍Step 4のフォーマットに準拠。

### 手順

1. フェーズの設定

   OKINYの場合、以下のフェーズで区切る：

   | フェーズ | 内容 |
   |---------|------|
   | 認知 | OKINYの存在を知る |
   | 興味 | 詳しく見てみる・登録を考える |
   | 登録・初回投稿 | アプリを入れて最初の投稿をする |
   | 継続利用 | 2回目以降、習慣化するまで |
   | 深化 | マイルストーン達成、好みの発見を体験する |
   | 共有・拡散 | 投稿カードをシェアする、人に勧める |

   ペルソナによってフェーズの重みが違う。記録型ペルソナは「継続」が長くなり、共有型は「共有」が早く来る。

2. ジャーニーマップの記述（各ペルソナ分）

   書籍p.44のフォーマットに沿って、フェーズごとに以下を書く：

   **行動** — そのフェーズで何をするか
   例：Xで友人がシェアした投稿カードを見る → App Storeで検索 → ダウンロード

   **思考** — そのとき何を考えているか
   例：「何これ？」「面白そう」「でもまたSNS増えるのか…」

   **感情** — ポジティブ/ネガティブ、その理由
   例：好奇心（＋）→ 面倒くさい（−）→ 投稿してみたら意外と簡単（＋）

   **接する情報・人・物** — そのフェーズで触れるもの
   例：投稿カード画像、App Store説明文、チュートリアル画面

   **気づき** — 設計者としてこのフェーズで気をつけるべきこと
   例：「App Storeの説明文で『3つ選ぶだけ』が伝わらないと離脱する」

3. 感情曲線を描く（各ペルソナ分）

   ジャーニーマップの下に、横軸＝フェーズ、縦軸＝感情（上がポジティブ、下がネガティブ）の折れ線を描く。

   書籍の指針：
   - 必ずしも全部ポジティブにする必要はない
   - ネガティブな「谷」がある方が、その前後の「山」が際立つ
   - 大事なのは、コントロールできる谷とできない谷を区別すること
   - コントロールできる谷は取り除く。できない谷は、前後に山を作って全体の体験を磨く

4. 離脱ポイントの特定

   感情曲線が下がっているポイントを全て抜き出し、以下を判定：
   - その谷はOKINY側でコントロールできるか？
   - できるなら対策案は？
   - できないなら、その前後で何ができるか？

5. ペルソナ間の比較

   2〜3人分のジャーニーを並べて：
   - 全ペルソナ共通の谷（＝最優先で対策すべき）
   - 特定ペルソナだけの谷（＝その層向けの施策）
   を区別する。

6. 既存の設計との突合

   事業計画書・要件定義書に書かれている機能（今日のお題、マイルストーン、スレッド、自己分析等）が、ジャーニーの各フェーズで適切に効いているか確認する。
   - 「この機能はどのフェーズの、どの感情に対して効くのか」を紐づける
   - どのフェーズにも対応する機能がない空白を見つける
   - 機能はあるがフェーズとの紐づけが弱いもの（＝UIの見せ方の問題）を見つける

### AIがやること
- ジャーニーマップのテンプレート生成（フェーズ×5項目の表）
- ペルソナ＋シーンデータを渡せばジャーニーの叩き台を生成（行動・思考・感情・接点・気づきを記入）
- 感情曲線の可視化（データを渡せばグラフ化）
- 離脱ポイントの抽出＋対策案の候補リスト提示
- ペルソナ間比較の差分表作成
- 既存機能との紐づけ表生成（要件定義書の機能リスト×フェーズのマトリクス）
- 空白フェーズの検出

### 人間がやること
- ジャーニーの叩き台に対する「ここは違う」の修正指示
- 感情曲線の上下の最終判断（「ここは本当にポジティブか？」の自問）
- 離脱ポイントの対策案の採用/不採用
- 既存機能の紐づけが「本当にここで効くか」の判断

- [ ] 感情曲線（各ペルソナ）
- [ ] 離脱ポイント一覧＋コントロール可否＋対策案
- [ ] 既存機能との紐づけ表（機能×フェーズのマトリクス）
- [ ] 空白フェーズ・弱い紐づけのリスト

## Day 5：Step 6 登場人物・小道具 → Step 7 デザイン原則＋22エッセンス

### 目的
ジャーニーの各フェーズに必要な「人」と「もの」を整理し、22の体験エッセンスとの突き合わせを経て、検証済みのデザイン原則を確定させる。

### 手順

1. タッチポイント一覧の作成

   ジャーニーの各フェーズで、ユーザーが接触するものを全て列挙する。

   【登場人物】（＝ユーザー以外の「人」）
   - 運営チーム（お題を出す人、ピックアップする人）
   - 他のユーザー（投稿を見る、スレッドに回答する）
   - 外部SNSの友人（シェアされたカードを見る人）
   - 公式アカウント（Phase 2以降）

   【小道具】（＝ユーザーが触れる「もの」）
   - App Storeの説明文・スクリーンショット
   - 投稿カード画像（外部SNSで見るもの）
   - チュートリアル/オンボーディング画面
   - 投稿フォーム（テーマ＋3つの入力画面）
   - タイムライン/ホーム画面
   - 今日のお題の通知
   - マイルストーン通知・ポップアップ
   - プロフィール画面（蓄積表示）
   - 検索画面
   - シェアボタン・シェアカード生成画面

2. フェーズ×タッチポイントの紐づけ

   表を作る：

   | フェーズ | 登場人物 | 小道具 | 目的（何のために接触するか） |
   |---------|---------|--------|--------------------------|
   | 認知 | 外部SNSの友人 | 投稿カード画像 | 「何これ？」を引き出す |
   | 興味 | — | App Store説明文 | 「使ってみようかな」に変える |
   | ... | ... | ... | ... |

   各セルで「この小道具は、このフェーズの感情曲線のどこに効くか」を確認。

   - **？ 未定** — 該当するか自体が不明。要検討

4. デザイン原則の確定

   Step 0の仮説棚卸しで「未検証」だったものの中から、Step 1〜6を経て「これは正しかった」と判断できたものだけを、デザイン原則に昇格させる。

   書き方のルール：
   - 「〜すべき」ではなく「〜する。なぜなら〜だから」の形式
   - 1原則につき1〜2文
   - 10個以内に収める
   - 各原則にUI上の具体的な帰結を1つ以上書く

   例：
   「投稿フォーマットは3つ固定にする。なぜなら制約があるから厳選が生まれ、選ぶ行為が自己対話になるから。→ 入力欄を可変にするUIは使わない」

   まだ検証が足りないものは原則にしない。「仮説のまま」と明記してリストに残す。

5. アイデアメモの判定

   最初の依頼にあったアイデアメモの各項目について、22エッセンスとの突き合わせ結果を踏まえて最終判定する：
   - Phase 1で実装する
   - Phase 2以降に回す
   - 不採用（理由を記録）
   - 要継続検討

### AIがやること

- 22エッセンス突き合わせ表の全量記入（OKINYの機能・アイデアとの照合、採用判断の案まで提示）
- デザイン原則の文章化（仮説棚卸し＋検証結果から原則案を生成）
- アイデアメモ判定表の生成（各項目の判定案＋根拠）

### 人間がやること
- 各エッセンスの採用/不採用の最終決定
- デザイン原則に「昇格させるか」の最終判断（「本当に検証されたか？」の自問）
- アイデアメモの取捨選択の最終決定

## Day 6：統合・検証

### 目的
Day 1〜5の成果物を通しで読み、矛盾・抜け・飛躍がないかを検証する。

### 手順

1. 全成果物の通し読み
   以下の順に読む：
   ペルソナ → ゴール → シーン → ジャーニー → タッチポイント → エッセンス → デザイン原則
   「この流れで論理が一本通っているか」を確認。

2. 矛盾チェック
   特に以下を確認：
   - ペルソナの不満 → ジャーニーで解消されているか
   - ゴールの優先価値 → デザイン原則に反映されているか
   - 離脱ポイントの対策 → タッチポイントに小道具が配置されているか
   - 22エッセンスの採用判断 → ジャーニーの具体的なフェーズに紐づいているか
   - デザイン原則 → 仮説棚卸しで「未検証」のまま昇格させていないか

3. 要件定義書との差分確認
   UX設計の結果、既存の要件定義書に追加・変更・削除すべき項目があるかを洗い出す。
   - 新たに必要になった機能
   - 優先度が変わった機能
   - 不要と判断された機能
   - フェーズの入れ替え

4. 修正作業
   矛盾や抜けが見つかった箇所を修正する。

### AIがやること
- 全成果物を渡せば矛盾チェックを実行（観点リスト＋発見した矛盾の一覧を出力）
- 要件定義書との差分抽出（成果物×要件定義書の自動照合）
- 修正案の提示（矛盾ごとに修正オプションを提案）

### 人間がやること
- 通し読みでの「違和感」の検知（論理ではなく直感で引っかかる部分）
- 矛盾への対応方針の最終決定（修正するか、許容するか）
- 修正内容の承認

### Day 6の成果物
- [ ] 矛盾チェック結果（発見した矛盾と対応方針）
- [ ] 要件定義書への差分リスト
- [ ] 修正済み最終版一式

## AIの使い方ガイド（全Step共通）

### 基本方針
作業は全てAIがやる。人間は判断と意思決定だけ。

### 進め方
1. 各Stepの冒頭で、前Stepの成果物をAIに渡す
2. AIがそのStepの全成果物の叩き台を生成する
3. 人間が目を通し、「ここは違う」「これでいい」を判断する
4. 修正指示を出せばAIが反映する
5. 最終承認して次のStepへ

### AIがやること（全Step共通）
- テンプレート・フォーマットの生成
- メモ・箇条書きから整った文章・表への変換
- 前Stepの成果物を入力として次Stepの叩き台を自動生成
- 仮説棚卸し・事業計画書・要件定義書との突合チェック
- 矛盾・抜けの検出と指摘
- 修正案・代替案の提示

### 人間がやること（全Step共通）
- インタビューの実施（対人は人間のみ）
- SNSの実際の閲覧（リアルタイムSNSはAI不可）
- グループ分け・優先順位づけの最終判断
- 採用/不採用の意思決定
- 感情曲線の上下判断
- 「この矛盾は許容するか」の決定
- AIの出力に対する「違う」「これでいい」の判定
