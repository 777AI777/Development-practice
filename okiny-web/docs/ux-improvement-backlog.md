- 最優先は「初回投稿完了までの摩擦削減」と「再訪導線」。
- `UI_mock.pen` 準拠を維持しつつ、無駄導線は開発フラグ側へ退避。

## 無駄構成の整理方針

- 検証用 `states/*` は削除済み。
- 本番導線の主CTAと検証ショートカットを同一行に置かない。
- 同一目的の重複ボタン（例: 設定画面の即時ログアウト+確認ログアウト）を統合。
